# 案例库右键删除菜单设计

## 目标

在案例库左侧参考图卡片上增加右键删除能力，同时保留现有删除入口，避免误删并兼容非鼠标用户。

## 交互

- 在任意案例卡片上右键，阻止浏览器默认菜单。
- 在鼠标位置显示轻量菜单，包含“删除案例”。
- 菜单显示当前案例标题。
- 点击“删除案例”后显示浏览器确认框。
- 用户确认后复用现有删除逻辑，删除案例和关联分析，并重新计算审美画像。
- 用户取消则不修改状态。
- 点击菜单外、滚动页面或按 Escape 时关闭菜单。
- 打开另一个案例菜单时替换当前菜单。

## 状态

新增非持久化模块状态：

```text
referenceContextMenu = {
  referenceId,
  x,
  y
}
```

菜单状态不写入 `localStorage`，页面刷新后自然消失。

## 可访问性

- 菜单使用 `role="menu"`。
- 删除按钮使用 `role="menuitem"`。
- 保留原有可点击删除入口，兼容触屏和键盘操作。
- 删除按钮使用危险操作配色，但不扩大为全卡片点击。

## 测试

- UI 源码包含 `contextmenu`、菜单钩子和关闭逻辑。
- 删除仍复用 `deleteReferenceFromState`。
- 右键本身不立即删除案例。
- 确认删除后案例数量减少，取消后不变。
- 点击空白处和 Escape 关闭菜单。
