# TasteOS 工作台重置设计

## 目标

在不改变现有业务逻辑和数据结构的前提下，把首页重置为可发布的 PC 端视觉分析工作台。页面必须完整保留案例库、审美画像、提示词工坊、结果反馈、产品说明五个模块，以及本地图片上传、图片链接、分析模式和开始分析功能。

## 布局

- 1440×900 为基准画布，最低桌面宽度 1200px。
- 顶部导航高 56px，左侧流程栏宽 72px。
- 案例库默认宽 320px，分析区弹性伸缩，配置栏默认宽 320px。
- 1280px 宽度时两侧栏压缩，但上传配置仍保持可见。
- 页面不改成移动端堆叠，不隐藏核心功能。

## 视觉

- 使用 TasteOS 冷雾灰背景、白色结构面板、1px 冷灰边框。
- 主色固定为 `#2F6BFF`，只用于主要动作、选中和焦点。
- 字体使用 `Inter, "PingFang SC", "Microsoft YaHei", sans-serif`；提示词和技术数据使用 JetBrains Mono。
- 普通面板无装饰阴影，仅中心分析画布允许轻微层级阴影。
- 圆角控制在 8–12px，使用稳定 8px 间距网格。

## 核心交互

- 案例卡片保持稳定缩略图比例，并清楚显示选中状态。
- 上传区支持本地图片、图片链接和即时预览。
- 分析进行中仅禁用配置表单，不阻塞导航和案例浏览。
- 五个一级模块始终可访问。
- 所有按钮和表单控件提供可见键盘焦点。

## 验收

- 1440×900 下同时看到案例库、分析结果、上传配置三栏。
- 1280×800 下上传配置仍可见且可操作。
- 本地文件输入、图片链接输入、模式选择和开始分析按钮均存在。
- 五个一级模块均存在。
- UI 回归测试和现有 Node 测试全部通过。
