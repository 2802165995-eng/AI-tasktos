# TasteOS 演示脚本

这份脚本用于 3-5 分钟项目演示。应用里不放内部说明，这份文档只作为演示前准备。

## 30 秒介绍

TasteOS 是一个个人审美提示词工坊。它解决的问题是：用户会收藏很多喜欢的海报、社媒封面和视觉参考，但很难说清楚自己喜欢什么，也很难把这种偏好转成稳定可复用的生图提示词。

TasteOS 的闭环是：

```text
参考图 -> 视觉分析 -> 审美画像 -> 场景化提示词 -> 外部生图工具 -> 结果反馈
```

## 3 分钟演示路径

### 1. 展示案例库

打开首页后先看案例库和当前分析卡。

演示重点：

- 这里不是普通图片收藏夹。
- 每张图都会被拆成用途、风格、构图、色彩、字体、情绪和负向提示词。
- 默认案例用于说明产品闭环，用户也可以删除并换成自己的参考图。

### 2. 导入自己的参考图

在左侧上传区：

- 可以填写标题，也可以不填。
- 粘贴图片链接或上传本地图片。
- 选择“真实模型分析”或“离线演示分析”。
- 点击“开始分析”。

演示重点：

- 真实模型分析会出现加载态，避免用户以为页面卡住。
- 如果真实模型失败，会回退到离线演示分析，保证演示不中断。
- 前端不会保存或暴露 API Key，真实分析通过本地代理完成。

### 3. 展示视觉分析结果

分析完成后看右侧结果：

- 反推提示词：如果要生成这张图，可能需要怎么描述。
- 提示词基因：可复用关键词。
- 用途和风格：由模型或离线规则自动识别。
- 负向提示词：后续生图时需要避免的方向。

演示重点：

> 这个产品不是判断图片好不好，而是把“我喜欢这种感觉”拆成可复用的创作语言。

### 4. 展示审美画像

切到“审美画像”。

演示重点：

- 多张参考图会聚合成长期偏好。
- 偏好分成版式、色彩、字体、情绪和需要避免的方向。
- 审美画像可以导出，作为后续创作或项目文档的一部分。

### 5. 生成提示词

切到“提示词工坊”。

操作：

- 选择使用场景，例如“校园活动海报”。
- 填写生成目标，例如“AI 创作分享会”。
- 选择模型格式，例如“即梦”“通义万相”或“豆包生图”。
- 点击“生成提示词”。

演示重点：

- 提示词不是从一句需求凭空生成，而是由“具体场景 + 审美画像 + 模型格式”共同生成。
- 输出包含正向提示词、负向提示词和可复用关键词。
- 可以分别复制正向、负向，也可以复制完整提示词包。

### 6. 记录结果反馈

切到“结果反馈”。

操作：

- 选择刚生成的提示词。
- 标记结果：喜欢、方向接近或不喜欢。
- 选择问题标签，例如“文字不可读”“画面太乱”。
- 保存反馈。

演示重点：

- 反馈不是为了做评价表，而是为了让失败经验沉淀进下一轮提示词。
- 这让 TasteOS 区别于一次性提示词生成器。

## 5 分钟深度讲法

可以按这个结构展开：

1. AI 生图工具越来越强，但用户的审美表达能力没有同步变强。
2. Pinterest / 花瓣解决收藏，ChatGPT 解决一次性文本生成，生图工具解决输出。
3. TasteOS 解决中间层：把个人审美偏好结构化，并转成可复用提示词资产。
4. MVP 收窄到活动海报、社媒封面、作品集封面，因为这些场景都有标题、主视觉、信息层级和传播目标。
5. 技术上把视觉分析抽象成稳定结构，离线分析和真实模型分析都能接入同一套前端流程。

## 主动说明的限制

建议主动说明，不要等对方追问：

- 当前是本地 MVP，没有用户系统和云端数据库。
- 应用不直接生图，而是把提示词复制到外部生图工具。
- 离线演示分析用于保证流程可跑，真实模型分析作为可选增强。
- 第一版聚焦平面视觉，不覆盖 UI、包装、视频或完整品牌系统。

## 结尾总结

可以这样收尾：

> 我做 TasteOS 的重点不是再做一个 AI 生图工具，而是把生图之前最模糊的“审美偏好表达”产品化。它把参考图变成审美画像，再生成场景化提示词，并通过结果反馈持续校准。
