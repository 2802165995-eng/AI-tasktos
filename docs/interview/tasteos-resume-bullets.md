# TasteOS 简历写法

## 项目一句话

TasteOS 是一个面向 AI 视觉创作者的个人审美提示词工坊，支持从海报/社媒封面/作品集封面等参考图中提取视觉偏好，生成个人审美画像，并输出适配多种生图工具的中文提示词包。

## 简历项目名

**TasteOS：AI 视觉创作者的个人审美提示词工坊**

## 推荐简历版本：产品 + 前端 + AI 应用

- 设计并实现 TasteOS 个人审美提示词工坊，围绕“参考图导入、视觉分析、审美画像、提示词生成、结果反馈”构建 AI 视觉创作前置工作流，帮助用户将模糊审美偏好转化为可复用创作指令。
- 抽象参考图分析结构，覆盖用途、风格、构图、色彩、字体、情绪、关键提示词和负向提示词等维度；支持离线演示分析与真实多模态模型分析共用同一套前端数据结构，降低后续 provider 替换成本。
- 接入本地后端代理 `/api/analyze-reference`，支持阿里云百炼 DashScope / OpenAI 视觉模型分析，API Key 仅保存在服务端环境变量，前端不暴露敏感信息；模型失败时自动回退离线分析，保证演示流程稳定。
- 实现提示词工坊，支持通用、Midjourney、DALL-E、Stable Diffusion、即梦、通义万相、豆包生图等格式适配，并提供正向提示词、负向提示词、完整提示词包的复制与降级兜底。
- 使用零依赖 HTML/CSS/JavaScript 完成可运行 MVP，支持本地图片/链接上传、案例删除、localStorage 持久化、提示词历史、审美画像导出、结果反馈和本地测试，验证从视觉参考到提示词资产的完整产品闭环。

## 首轮可用性走查后的推荐版本

**TasteOS｜AI 审美分析与 Prompt 策略工作台｜独立产品项目**

- 针对 AI 生图用户“会收藏参考图、但难以描述和稳定复现审美”的问题，完成用户场景拆解与 MVP 定义，将产品定位为生图前的审美表达与 Prompt 策略工作台。
- 设计“参考图导入—视觉分析—审美画像—场景化 Prompt—结果反馈”闭环，规划案例库、审美画像、提示词工坊、结果反馈等 5 个核心模块，将分散的视觉偏好沉淀为可复用审美资产。
- 抽象用途、风格、色彩、构图、提示词基因和负向提示词等结构化分析字段，支持离线规则与真实多模态模型共用同一数据结构，并适配 Midjourney、DALL-E、Stable Diffusion、即梦、通义万相、豆包生图等格式。
- 完成首轮端到端可用性走查：导入参考图后案例库由 13 增至 14，成功生成并复制场景化 Prompt，提交 1 条结果反馈后画像聚合状态同步更新，验证跨模块数据流与基础偏好校准机制。
- 主导产品定义、功能拆解、交互方案与验证，借助 Codex 完成前端 Demo 和多轮 UI 迭代，落地支持图片上传、模型分析、画像导出、Prompt 历史和反馈记录的可运行 PC 端 MVP。

## 偏产品经理版本

- 从 AI 生图用户“会收藏参考但难以复现审美”的痛点出发，定义 TasteOS 的 MVP 范围，聚焦活动海报、社媒封面、作品集封面三类高频平面视觉场景。
- 设计“参考图 -> 视觉分析 -> 审美画像 -> 场景化提示词 -> 结果反馈”的产品闭环，将一次性提示词生成转化为长期可复用的个人审美资产。
- 明确产品边界：不直接做生图平台，不做素材站，不做完整品牌系统，而是专注生图前的审美表达和提示词准备层。
- 通过真实模型接入、国内外生图工具格式适配、复制失败兜底和演示文档完善，提升 MVP 的真实可用性和可展示性。

## 偏前端/工程版本

- 使用原生 HTML/CSS/JavaScript 构建零依赖单页应用，拆分 `app.js`、`tasteEngine.js`、`promptAdapters.js`、`apiClient.js`、`clipboard.js` 等模块，保持 UI、领域逻辑、模型适配和复制能力边界清晰。
- 设计真实模型代理服务，前端通过 `/api/analyze-reference` 请求分析结果，服务端读取 DashScope / OpenAI 环境变量并统一返回结构化分析数据。
- 实现浏览器剪贴板降级方案：优先使用 Clipboard API，权限受限时展示手动复制文本框，避免演示或真实使用时被浏览器权限中断。
- 编写多组本地测试覆盖视觉分析、提示词适配、复制降级、真实模型请求构造、状态管理、文本完整性和导出逻辑，保证 MVP 快速迭代下的可回归性。

## 面试 30 秒开场

我做 TasteOS 是因为 AI 生图工具越来越强，但很多用户的问题不是没有工具，而是不知道如何稳定表达自己的审美。用户可能收藏了很多喜欢的海报和封面，却很难把“我喜欢这种感觉”转成可复用提示词。

所以我做了一个生图前的审美表达层：用户导入参考图后，系统分析用途、风格、构图、色彩、字体和情绪，聚合成个人审美画像，再按具体场景和生图工具格式生成中文提示词包。当前版本已经支持离线演示分析和真实多模态模型分析，并能输出即梦、通义万相、豆包生图等格式。

## STAR 讲法

**Situation**：AI 生图工具降低了视觉创作门槛，但提示词质量和审美表达成为新的瓶颈。很多用户会收藏参考图，却难以复现稳定风格。

**Task**：我希望做一个不直接生图、而是帮助用户沉淀审美偏好并生成可复用提示词的中间层工具。

**Action**：我先写 PRD，将 MVP 收窄到活动海报、社媒封面和作品集封面；再设计案例库、视觉分析、审美画像、提示词工坊和结果反馈五个模块；最后实现零依赖 Web MVP，并接入 DashScope / OpenAI 真实视觉分析代理。

**Result**：项目已跑通参考图导入、真实/离线分析、审美画像、中文提示词生成、多模型格式适配、复制兜底和反馈闭环，并通过本地测试与浏览器端到端验证。
